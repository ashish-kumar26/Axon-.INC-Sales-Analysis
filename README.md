(MS Excel | SQL | Power BI | Tableau | Python)
________________________________________
📌 Project Overview
This project focuses on analyzing CRM sales data to understand how leads are generated, how they convert into opportunities, and how those opportunities contribute to revenue.
The analysis follows a complete analytics workflow, starting from raw CRM tables and ending with business dashboards and predictive insights.
The project was completed using MS Excel, SQL, Power BI, Tableau, and Python, ensuring that the same KPIs and logic are consistent across all tools.
The main goals of this project are:
• Analyze lead generation performance
• Understand lead conversion into opportunities
• Evaluate sales pipeline performance
• Calculate expected revenue from opportunities
• Identify key factors affecting deal success

🧠 Business Logic & KPI Definitions
All KPIs were created using clear business logic to ensure consistency across Excel, SQL, Power BI, Tableau, and Python.
Total Leads
Total number of leads generated.
Converted Leads
Leads that were successfully converted into opportunities.
Conversion Rate
Percentage of leads converted into opportunities.
Conversion Rate =
Converted Leads / Total Leads
________________________________________
Total Opportunities
Total deals created in the CRM system.
________________________________________
Won Opportunities
Deals that were successfully closed.
________________________________________
Win Rate
Percentage of opportunities that were won.
Win Rate =
Won Opportunities / Total Opportunities
________________________________________
Expected Revenue
Estimated revenue based on opportunity probability.
Expected Revenue =
Opportunity Amount × Probability
This metric helps businesses estimate future sales performance.

🧮 SQL Analysis
SQL was used to work directly with the CRM tables and calculate important business metrics.
Key tasks performed:
• Cleaning numeric fields such as amount and probability
• Removing symbols like $ and % using REPLACE functions
• Calculating expected revenue using probability logic
• Aggregating metrics using SUM, COUNT, and CASE statements
Example logic used in SQL:
Expected Revenue = Amount × Probability / 100
SQL acted as the base validation layer to ensure results are correct before creating dashboards.
